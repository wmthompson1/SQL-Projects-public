Using the VISUAL BTS Labor Transaction Processor

The VISUAL ALTS Labor Transactions window is not available from the main Infor VISUAL menu. To make accessing the application easier, set up a short cut icon on your desktop or use the VISUAL Personal Menu Toolbar.

From the directory where your executables are installed, double-click Vmbtslab.exe.

Notice the rather simple appearance of the window. Because this window serves as a transaction processor only, it does not contain the same features as the VISUAL Barcode Labor Ticket Entry window. If you currently use the Barcode Labor Ticket Entry window and are interested in VISUAL ALTS as a replacement, contact your Infor Global Solutions Partner.

Both the Pending Transactions and the Completed Transactions table windows contain the following columns:

ID - The ID of the transaction.

Transaction Type - The type of transaction, one of the possible seven. See the beginning of the section.

Transaction Date - The date on which you performed the transaction.

Employee ID - The ID of the employee involved in the labor transaction.

Bad Quantity - The quantity bad reported from an operation.

Quantity - The quantity of the transaction.

Setup Completed? - Is the setup completed? Either Y or N appears.

Indirect ID - The Indirect ID of the labor transaction.

WO Base ID - The work order’s Base ID. This is the work order from which the employee is reporting labor. The work order must have a status of Released before it is eligible to have labor reported against it.

WO Lot ID - The work order’s Lot ID.

WO Split ID - The work order’s Split ID.

WO Sub ID - The work order’s Sub ID.

Operation Sequence Number - The sequence number of the operation in the work order.
