One Part at a Time Firming Planned Orders

Both Master Schedule and MRP items use planned orders to impose demand on lower level parts. Therefore, MRP explodes the master schedule for Master Scheduled parts and nets the demand of MRP parts to obtain planned orders. Lower level material requirements are attached to each planned order. Parts use these lower level material requirements in generating their plan.

The Planning Window currently explodes the current engineering master when creating planned orders during an MRP run. The same approach is taken when firming an existing planned order into a work order; for example, the current engineering master is copied. There is the potential for subtle differences between the time the planned order was generated and the firming action takes place.

The Planning Window (in repetitive mode) provides a way for the user to firm multiple planned orders with a single command. You indicate the date through which planned orders will be firmed by clicking and selecting a column in the display. All planned orders not yet firmed are displayed. All selected planned orders are converted to either work orders or purchase orders.

Because multiple orders imply multiple settings for some user input items needed for the firming process, the presentation of orders for firming are changed to a columnar format by VISUAL. Each row of the grid shows a single order about to be firmed. You may change some specific information when appropriate. In general, there will be defaults for every item.

Use the Type column when the part is either fabricated or purchased. The default is purchased. When the part does not have this property the column does not show. Further, if the part is not purchased the Vendor column is not shown.

The Vendor column is populated with the current primary vendor of the part. Want date is the want date of the planned order (as generated by MRP). Release date is the want date minus the standard leadtime. Qty is the quantity of the planned order (as generated by MRP). The order number column is automatically set.

The status column defaults to Firm. This causes VISUAL to be ready to generate a traveller for a work order. If the user sets this to Released the traveller is not automatically printed. For purchase order, a firm setting means the acknowledgment is automatically printed, while the released setting does not print the acknowledgment.

Automatic setting of the order number is done using one of the following methods.

  • Next number in sequence using the next-number-generator.

  • Use the want date in combination with the first operation resource ID.

  • Use the want date in combination with the part number.

In any case, the size limitation of the work order ID and purchase ID is imposed by the database design and cannot be exceeded. If this happens the system will report an error an abort the firming procedure.

The want date is combined with either the resource ID or part ID by concatenating the ID followed by a dash followed by the date without any punctuation. For instance, "PARTABC-01061997", or "MACHINE001-01061997." Again, if this results in an invalid ID or the order number already exists, the system complains and aborts the procedure.
