Working with Evaluation Sequences

Using the Re-evaluation list in the Step Properties dialog box, you can allow VISUAL to restart a workflow each time a document (for example, a Purchase Order or Purchase Requisition) is saved. Because a workflow step may have multiple paths, you can define the sequence in which VISUAL evaluates each path. As shown in the following example, the value in the Evaluation Sequence field determines the evaluation order, starting with Sequence 1 and ending with Sequence 3.
