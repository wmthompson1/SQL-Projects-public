Specifying Calendar Exception Days

In some instances, you may need to change a normal shift day, such as for a holiday. Use Calendar Exceptions to define changes to the standard schedule.

If you have multiple sites, you can import the calendar exceptions in Application Global Maintenance into the calendar exceptions you define in Site Maintenance.

  1. Complete one of these steps:

  • To define calendar exceptions at the tenant level, select Admin, Application Global Maintenance.

  • To define calendar exceptions at the site level, select Admin, Site Maintenance.

  1. Select Maintain, Calendar Exceptions.

  2. Click Insert.

  3. Specify this information:

Start Date - Specify the date the exception starts.

End Date - Specify the date that the exception ends.

1st Shift Start - Specify the time that the first shift starts.

Shift 1, Shift 2, Shift 3 - Specify the shift duration for the exception. If the shift is not to work at all, enter zero (0) in that shift’s column.

  1. Click Save. The updates that you made take effect the next time that you run the Global Scheduler.
