Specifying Customer Language Forms

Because some international companies have requirements for issuing documentation in the home language of the company with which they are dealing, you may need to be able to select from multiple languages when dealing with different aspects of international customer orders. For example, a situation may call for you to produce an order acknowledgement in German, a packlist in French, and an invoice in Italian. Use the Languages tab to set default Sold To and Bill To Language IDs for customers. Only uses these IDs if a customer in an international customer. If a customer is a domestic trading partner, there is no need to specify this information.

  1. With the customer for whom you want to specify Language IDs displaying in the Customer Maintenance window, click the Lang Forms tab.

  2. Click the Language ID (Sold To) button or arrow to select from a list of valid Language IDs. If no Language IDs appear in the Languages dialog box or drop down box, you have not added any Language IDs to the database. See Entering Languages.

This is the Language ID you want to associate with the customer’s main address.

  1. Click the Language ID (Bill To) button or click the arrow to select from a list of valid Language IDs. If no Language IDs appear in the Languages dialog box or drop down box, you have not added any Language IDs to the database.

This is the Language ID you want to associate with the customer’s billing address.

  1. Click the Save button to commit the Language IDs to this customer.
