Adding General Formula Information

By default, when you access the Formulas window, the General tab information appears for the current Formula ID. If the General tab information is not the topmost tab, click the General tab.

To add General tab information:

  1. From the drop-down list, select the Formula Type for the current Formula ID. Make your selection from: Commission, Discount, Global, Price, or Terms.

  2. From the Context drop-down, select the Context for the formula. The Context selection determines where the formula will be applied. Make your selection from: Commodity, Customer, Customer/Part, Customer Type, Customer/Product, Customer Type/Product, Global, Market, Market/Part, Part Group, Price Group, or Product. Your Context selection affects the fields that appear on the General tab.

  3. Enter or select the default Currency identification code to apply to the current formula. You can enter the Currency ID manually, or select it from the Currency table by clicking the Browse button.

  4. Enter or select the appropriate values, as determined by your Context selection. The possible fields that appear include:

Commodity Code - Applies to Commodity Code Context only. Enter or select the Commodity Code ID to which to apply the current formula.

Customer ID - Applies to Customer, Customer/Part, and Customer/Product Contexts. Enter or select the Customer ID to whom the formula applies.

Ship-to ID - Applies to Customer, Customer/Part, and Customer/Product Contexts. Enter or select the identification code for the location where material will be shipped. This allows you to have different formulas applied to different locations, even for the same customer.

Part ID - Applies to Customer/Part, Market/Part, and Part contexts. Enter or select the identification code for the part to which the current formula applies.

Customer Type - Applies to Customer Type and Customer Type/Product contexts. Enter or select the identification code for the type of customer to which you want to apply the formula.

Product Code - Applies to Customer/Product, Customer Type/Product, and Product contexts. Enter or select the identification code for the Product Code to which to apply the current formula.

Market ID - Applies to Market and Market/Part contexts. Enter or select the identification code for the Market to which you want to apply the current formula.

Part Group ID - Applies to Part Group context only. Enter or select the identification code for the Part Group to which you want to apply the current formula.

Price Group ID - Applies to Price Group context only. Enter or select the identification code for the Price Group to which you want to apply the current formula.

You can enter the values manually or select them from their respective browse tables by clicking the associated Browse buttons.

  1. In the Seq # field, enter the Sequence Number for the current formula. If you use multiple formulas for one price calculation, you can set the order in which the current formula is calculated. The Sequence Numbers are read in ascending order (from lowest to highest).

  2. In the Effective Date field, enter the date on which the current formula comes into effect. Prior to the Effective Date, the formula is not applied. Enter the date in Windows system format.

  3. In the Expire Date field, enter the date after which the current formula becomes inactive. After the Expire Date, the formula is not applied. Enter the date in Windows system format.

    If you do not want to have a date range, leave the Effective and Expire Date fields blank.

  1. Select the Active check box if you want to apply the current formula immediately. If the box is cleared, the current formula is not applied, even if it falls within the Effective and Expiration Dates.

  2. On the main VISUAL Price Book toolbar, click the Save button to save changes to the current formula.
