Netting By Required Date

Select Net By Required Date from the View menu to sequence demand and supply orders together by Date Required and Due Date. They are only aligned if they fall on the same date.

Netting by required date produces a picture of available quantity on each consecutive date. Exceptions indicate if a stockout or overstock condition exists on a given date, and suggest dates to release new supply orders.

You may find netting by required date easier to work with for materials that you maintain as inventory and issue as needed, because the focus is on each day’s total projected on-hand quantity versus demand.

Projected Quantity for Netting by Required Date

This is the projected on-hand quantity at the "end of the day," after all supplies have been received and demands have been met.

Exceptions for Netting by Required Date

When multiple supply and demand orders fall on the same date, you need only pay attention to the last exception for that date, since it applies to the final projected quantity for that date.

Overstock - Projected Quantity for the date is greater than or equal to the total Required Quantities of all demand orders, if any, for the same date. The day will end with a surplus of stock.

Stockout - Projected Quantity for the date is less than the total Required Quantities of all demand orders for the same date. Or, there are no demand orders and the Projected Quantity is zero or negative.

Rel. Upcoming / Rel. Now / Rel. Past Due - VISUAL is suggesting a release date for a new supply order due on this date, because the projected balance for the current date isn’t sufficient to meet the demand order for the date. These messages indicate that the suggested release date is close at hand, or already passed.

Rel. Past Due indicates that the suggested release date has already passed. Rel. Now indicates that the suggested release date is the current date. Rel. Upcoming indicates the suggested release date is within 3 days from the current date.

If suggested release date is more than 3 days away, the Stockout exception message is used.

Order Late - The Due Date of the supply order is earlier than the current date. The order is late. This exception only appears for supply orders with no aligned demand orders. This overrides any other exception that might be shown for the order.

Suggested Release Date

When there is a demand on this line that the planned supply cannot meet, VISUAL presents a suggested release date of a supply order to meet the demand. This is done by subtracting the Lead Time for the part (purchased or fabricated) from the required date for the demand.

For a purchase order, this is the date you should place the order for the part. For a work order, this is the date the order should be released.

Suggested Release Date uses the part lead time to back off a release date for a work order. You should consider this when setting the lead time for a fabricated part. The Global Schedule is not examined to determine whether the lead time can be met with the currently planned shop load. If you choose to place a work order based on this date, you should then use Check Availability in the Manufacturing Window to assess whether this produces a finish date that will meet the demand requirement date. You should then use Schedule Current Work Order, or run the Global Scheduler, to add the new work order into the schedule.

This field applies to both types of netting, MRP and by-date.

Modifying Supply Orders

You can change three fields for supply orders. This makes it easy to adjust orders without having to open Purchase Order Entry or the Manufacturing Window. Keep in mind that these actually change the purchase order or work order.

Due Date - This modifies the Desired Recv Date of the purchase order line item, or the Want Date of the Work Order.

Release Date - This modifies the Order Date of the purchase order, or the Release Date of the Work Order.

Status - This modifies the status of either order. Valid status values are:

 U Unreleased (Work Orders Only)

 F Firm

 R Released

 C Closed

 X Canceled

To modify the fields, make the changes as necessary, then click Save. The screen automatically refreshes to show the results of the change.

You can not modify these fields for a planned order generated by MRP. You must make it into an actual order first, using the Firm/Release Planned Order command from the Edit menu.
