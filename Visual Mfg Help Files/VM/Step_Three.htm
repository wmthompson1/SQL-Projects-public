Step Three

Run Costing Utilities and Prepare Manufacturing Journals

The more often you run costing utilities during the period, the quicker the costing process will be. We recommend running Receipt Transaction Costing and Inventory Transaction Costing (the top two options in the Costing Utilities window) daily. We also recommend running journal preparation and posting the journals at least once a week. When you are ready to close the period, use the following procedures.

Running the Costing Utilities

To access Costing Utilities:

  1. From the Eng/Mfg menu or from the Ledger menu, select Costing Utilities. The Costing Utilities window appears.

  2. In the top section, select all options in the top and bottom portions of the window.

  3. Click the Run button. The costing utilities analyze your database and cost transactions as appropriate.

When the costing utilities have finished running, a dialog box appears alerting you to the actions that the costing utilities took.

  1. If costing encounters errors, VISUAL prompts you to examine the errors in the error log.

  2. Click Yes to examine the log. The log appears.

The log identifies specific issues that interfered with the successful completion of costing. You must correct the problems identified in the log. If you do not correct the errors, they will compound over time.

If you do not want to review the log, click No.

The error log is stored in the directory where your executables are installed as VMPRJCST.log. You can review the log at any time.

  1. After you correct any errors in the error log, run the costing utilities and prepare the journals again.

Reviewing Journals

After costing has run to completion with no errors, prepare the G/L Transaction Reports to view the transactions before you post them.

To access the reports:

  1. From the Ledger menu, select Post Manufacturing Journals. The Post Manufacturing Journals window appears.

  2. From the Reports menu, select G/L Transactions. The Print G/L Transactions window appears.

  3. In the Print Selection box, select By Accounting Period.

  4. In the End Date of Acct Period field, enter any date that falls within the period you are closing. VISUAL automatically populates the field with the last day of the period.

  5. In the Sequence box, select Order ID/Trans, G/L Acct ID.

  6. In the Transactions box, select Print Unposted Txs. The transactions you generated by running the costing journals will appear in the report.

  7. In the Journal Type box, begin by selecting Purchases. After you run the Purchases report, run reports for each journal type except for Projects. See Reviewing Project Transactions, below.

  8. Click Ok. The report outputs.

  9. Compare this report to the reports you printed in step two to see if transactions costed as anticipated.

Reviewing Project Transactions

To review project transactions before you post them, run the G/L Transaction Report from the Project Summary window.

  1. From the Project menu, select Project Summary. The Project Summary window appears.

  2. From the File menu, select Print G/L Transactions. The Print G/L Transactions window appears.

  3. In the Print Selection box, select By Date Range.

  4. In the Starting Date field, enter the first date of the period you would like to close. In the Ending Date field, enter the last day of the period you would like to close.

  5. In the Transaction Type field, select Print All. The report will show all project-related transactions.

  6. Clear the Current User Only check box. The report will show all project-related transactions entered by any user.

  7. Select Include Rev Recognition to include revenue recognition transactions in the report.

  8. Select Include Rev Recg WIP Only to include only the WIP relief transactions on the report.

  9. In the Transactions box, select Print Unposted Txs to view the transactions generated by the costing utilities.

  10. Select a sort order for your report in the Sequence box.

  11. In the Mode box, select Inquiry Only to view the information in the table. Select Report Only to view the results in a report.

  12. Click Ok. The report outputs.
