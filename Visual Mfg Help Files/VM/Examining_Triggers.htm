Examining Triggers

To examine database triggers:

  1. Select Admin, Examine Triggers.

  2. Click the Run toolbar button.

This information is displayed in the header panel:

System Version – The current version of VISUAL

Database Brand and Patch Level – The type of database engine and patch level of the database appears in the Database Brand and Patch Level fields.

Database Version – The version of the database engine.

Total Triggers Expected – The total number of triggers expected in the database.

Total Triggers Found – Total number of triggers existing in the database.

This information is displayed in the results panel:

Missing Triggers – The names of required triggers that are missing from the database.

Additional Triggers – The names of any additional triggers that were found.

Disabled Triggers – The names of disabled triggers.

Audit Triggers – The names of the audit triggers that have been created. Because users can add Audit Triggers at their discretion, the entries listed in the Audit Triggers field are informational only.

These totals are displayed:

  • Total Triggers Missing

  • Total Additional Triggers

  • Total Disabled Triggers

  • Total Audit Triggers

  1. After you finish examining database triggers, click the Exit toolbar button.
