VISUAL BTS Purchase Receipts Entry Table

The Pending Transaction portion of the window contains the following columns; the Completed Transaction window contains the same columns with the addition of column Transaction ID. After VMBTSBI has successfully processed the transaction, its ID appears in this column.

ID - The ID of the transaction.

Trans Type - The type of the transaction.

Trans Date - The date on which you scanned in the purchase order.

Purchase Order ID - The ID of the purchase order that you are receiving. See Purchase Order Entry for more information.

Line Number - The number of the line that you are receiving as it appears in the purchase order.

Quantity - The quantity of the purchaser order line.

To Warehouse - The ID of the warehouse where you are receiving the goods.

To Location - The ID of the location in the warehouse into which you are receiving the goods.

Base ID - The Base ID of the work order - if there is one - to which this purchase receipt line is allocated.

Lot ID - The Lot ID of the work order - if there is one - to which this purchase receipt line is allocated.

Split ID - The Split ID of the work order - if there is one - to which this purchase receipt line is allocated.

Sub ID - The Sub ID of the work order - if there is one - to which this purchase receipt line is allocated.

Operation Sequence Number - The operation sequence number of this material in a work order.

Requirement Piece Number - The requirement piece number of the material in a work order

Customer Order ID - The ID of the order that you are now receiving. This comes from the packlist for the order.

Customer Order Line Number - The line number of the order that you are now receiving. This comes from the packlist for the order.

User ID - The ID of the user who is currently scanning.
