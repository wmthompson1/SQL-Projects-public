Adding Markup Matrix Table Information

After you select the appropriate information on the Markup header section of the Markups window, you can set up price breaks on the table.

To add Markup Matrix information:

  1. On the VPB toolbar, click the Insert Line button. This activates the next available field on the Markup Matrix table. The context-based ID, Break Type and Currency ID are automatically inserted on the table.

  2. The information on the table varies, based on the Context, Break Type, and Show Effectivity selections. Enter the information in the table as follows:

Commodity Code - Applies to Commodity Context. The value of the identification code entered or selected in the Commodity Code field is copied. Commodity Codes are numbers assigned to products, defined by the industry to be commodities (interchangeable from various manufacturers).

Customer ID - Applies to Customer, Customer/Part, and Customer/Product Contexts. The identification code entered or selected in the Customer ID field is copied. Customer IDs are used throughout the applications to identify clients to whom you ship products and materials.

Ship-to ID - Applies to Customer, Customer/Part, and Customer/Product Contexts. The identification code entered or selected in the Ship-to ID field is copied. The Ship-to ID allows you to see the destination for the customer merchandise.

Part ID - Applies to Customer/Part, Market/Part, Part, and Part Group Contexts. The identification code entered or selected in the Part ID field is copied. Part IDs are used throughout the system to identify items and merchandise you manufacture or sell.

Customer Type - Applies to Customer Type and Customer Type/Product Contexts. The identification code entered or selected in the Customer Type field is copied. Customer Types are used to classify the companies with whom you do business.

Product Code - Applies to Customer/Product and Customer Type/Product Contexts. The identification code entered or selected in the Product Code field is copied. Product Codes serve as a product classification within which you can assign additional information, including account information.

Market ID - Applies to Market and Market/Part Contexts. The identification code entered or selected in the Market ID field is copied.

Part Group ID - Applies to Part Group Context only. Part Groups are used to classify part IDs. The identification code entered or selected in the Part Group ID field is copied.

Qty - Applies to a Quantity Break Type. Enter the quantity for which you want to set the breaks or differences in prices for the selected Context. For example, you may run a "Buy 20 and receive a 5% discount" promotion.

Amount - Applies to an Amount Break Type. Enter the value for which you want to set the differences in prices for the selected Context. For example, you may have a promotion that states: "10% off when you purchase $100 or more".

Currency ID - Identification code for the international Currency for the current price matrix. Multiple (international) currencies are supported. By default, the entity currency is inserted. To change the Currency ID on the table, select a different Currency in the header portion of the Prices window.

Unit Cost - Enter the price per unit of measure for the current Context. Maintain Units of Measure outside of VISUAL Price Book.

Effective Date - If you have the Show Effectivity check box selected within the header of the Prices window, VISUAL Price Book prompts you for the date in which the price break comes into effect. The price breaks do not apply prior to the Effective Date. Enter the Effective Date in Windows system format.

Expire Date - If you have the Show Effectivity check box selected within the header portion of the Prices window, VISUAL Price Book prompts you for an expiration date for the current price break. Price breaks do not apply beyond the Expire Date. Enter the Expire Date in Windows system format.

You can leave Effective and Expire Date information out of a particular row within the table. If you enter one of the dates, you must enter the other end of the date range.

  1. On the VISUAL Price Book main toolbar, click the Save button to save changes to the current record.
