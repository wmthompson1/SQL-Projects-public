Generating Deadline & Recurring Events

You can set Message Maintenance to alert you when a deadline for a certain event has arrived. You can also set Message Maintenance to alert you of recurring events.

The following is an example of a deadline event for a customer order.

  1. First set up the event.

Enter or select the following information from the Event Maintenance window:

    • Class: Deadline event

    • Action: Popup

    • Status: Active

    • Type: Customer order

    • Base ID: CO-001

The Do Until field in the Frequency section is now grayed out, but the date field allows an entry.

  1. Enter a deadline date in the Do Until field.

  2. Under the Message Tab enter:

Customer order %ID Desired ship date %1 needs immediate attention as of %2.

  1. Under the Query Tab enter:

select id from customer_order where id=%BASE_ID

and status not in(‘C’,’X’)

and desired_ship_date<=SYSDATE INTO %BASE_ID

  1. Under the Tokens Tab enter:

select desired_ship_date,SYSDATETIME from customer_order

where id=%BASE_ID

  1. Click the Save button to save the event.

  2. Now go to Customer Order entry and create order CO-001, with the Desired ship date set to the deadline date.

Enter any part, quantity, vendor, and price.

  1. Save the customer order.

  2. Now return to Message Maintenance and select Process Events from the File menu.

  3. Click the Generate Deadline/Recurring Events button.

VISUAL produces the proper message for the deadline event. If the system date is after the desired ship date, the system automatically generates the proper message every time you log on to VISUAL.
