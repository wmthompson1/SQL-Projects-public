Fixing Query Error Events

After you have viewed the list of events that contain errors and have viewed the error messages, return to the Event Maintenance window to correct the error. In the example above with the invalid SQL statement, return to the Tokens tab and correct the problem; in this case, a simple spelling mistake.

Now return to the Process Events dialog box to execute the event with the error that you just corrected.

  1. Select Process Events from the File menu.

The Process Events dialog box appears.

  1. Select Error from the Status drop down menu.

A list of events with errors appears in the table. You can scroll to the right of the table to see the error message for the event.

  1. Select the event that you just corrected.

A dialog box appears, asking you if you want to execute the displayed query.

  1. Click Yes to execute the query with your recent changes.

A check mark appears in the column to the left of the Event ID column.

  1. Click the Save button below the table of events.

The check mark disappears.

  1. Click Ok to close the Process Events dialog box.

To see if VISUAL has processes your corrected, open the Process Events dialog box again and look for the event you corrected in the Completed events table.
