Creating a New Workflow Template

Workflow templates allow you to select the VISUAL application for which you want to create a workflow.

To create a new Workflow template:

  1. In the Open Existing Workflow dialog box, click the New button.

The Create New Workflow dialog box appears.  

  1. In the Type list, select one of the following:

Master - This feature is not available at this time.

Predefined - Starts with an empty template.

User Defined - This feature is not available at this time.

  1. In the Application Area list, select the application area (for example, A/P Invoice Entry or Cash Book) of VISUAL for which you want to create a workflow.

  2. In the Template Name field, type the name you want to use for this template.

  3. In the Usage list, select one of the following:

All Documents - VISUAL applies this template each time you create a workflow.

Single Document - VISUAL uses this template for a specific document. For example, you can create a custom template for a specific purchase order or quote.

  1. Do one of the following:

    • If you selected All Documents in the Usage list, go to the next step.

    • If you selected Single Document in the Usage list, type the exact document ID (for example, enter the order ID if you want to create a workflow for a specific purchase order); then click the OK button.

The Associated DB Table field is "view only" and lists the database tables associated with the VISUAL application that you selected in the Application Area list.

  1. In the Status list, select one of the following:

Active - The template is active within the VISUAL logic. When a template is active, the VISUAL application you selected in the Application Area list (for example, A/P Invoice Entry or Cash Book) behaves in accordance with the workflow and its set of rules.

Inactive - The template is not active within the VISUAL logic.

  1. Click the Ok button.

VISUAL creates a new Workflow Template.

To build a workflow, refer to the Building Workflow Templates.
