Creating the Header Card

To create a new project using the Project window:

  1. In the Project window dialog box, click the New button.

The Project Header dialog box appears.

  1. If you want to associate this project with a part, click the Part ID button and select the part to which you want this project to apply.

The description for the part you selected automatically appears in the appropriate field.

Typically, the standard project structure does not include a Part ID. If you enter a part ID, VISUAL populates part default information in the Drawing Revision, Drawing File, Product Code, and Commodity Code fields.

  1. Click the Project Info button and select the Project ID you want to use for this project.

The appropriate project information and Warehouse ID automatically appear.

You must select a project ID that has not already been used in a project header. If you select an ID that has already been used, you will be alerted that you cannot use the ID when you attempt to save the record.

If you enter a new ID in the Project Info field, when you tab out of the field the system will ask you if you want to create the project. Click Yes to create the project in the Project Maintenance window.

  1. Use the Calendar buttons to select Release and Want dates.

The Completed By Date automatically appears in the Want Date field.

  1. Click in the Quantity field and enter the quantity of this product for which you are creating this project.

  1. Click the WBS button and select the work breakdown structure you want to use for this header.

If you want to enter any specification notes about this project, click in the Specifications field and enter the information about this project you want.

As you select various information in the Project Header dialog box the appropriate information appears in the General tab.

  1. Enter the following information on the General tab:

Project Manager – Enter the name of the manager of this project in the Project Manager field.

Engineered Date – Use the Calendar button to select the date on which you engineered this product.

Drawing Number – Enter the drawing number in the Drawing Number field.

Drawing Revision – Enter the drawing revision number in the Drawing Revision field.

Drawing File – If you want to include a drawing file for this project, click the navigation button and select the file you want to use.

If you have selected an AutoCad® drawing, you can view but not alter that drawing by clicking the View button.

Entered By – The method used to create the project is displayed. This field is populated by the system. For more information, see Working with Information in the Entered By Field.

Product Code – If you want to include a Product Code for this project, click the arrow and select the appropriate Product Code from the list.

Commodity Code – If you want to include a Commodity Code for this project, click the arrow and select the appropriate Commodity Code from the list.

Traveller Printed – If you have previously printed a traveller for this project, the appropriate date and time appears in the Traveller Printed field.

Forward Schedule from Release Date – If you want the Scheduler to work forward from the release date, scheduling the work order for completion as soon as possible, select the Forward Schedule from Release Date check box.

If you do not select the Forward Schedule option, the Scheduler attempts to schedule the work for completion by its want date and the want dates of all other work orders.

Treat Release Date as Hard – If you want the Concurrent Scheduler not to attempt any scheduling of the work order until the release date, select the Treat Release Date as Hard check box.

Blanket Order - Select this check box if the part associated with the project is part of a larger, blanket order. In a blanket order, the part received can be used in any of a number of projects or non-project transactions.

If you want to enter any User Defined information, click the User Defined tab and enter the appropriate information.

Click the appropriate button and enter the following information:

Doc Ref – If you want to associate documents with this project, click the Doc Ref button.

Pict/Object – If you want to associate a graphic with the project, click the Pict/Object button.

Macros – Macros are small script-based programs that can make the most complex tasks simple by automating time-consuming processes and eliminating the chance of incorrect data entry.

Project T&C – Click the Project T&C button to review the terms and conditions for this project.

Customizable UDF’s – To add additional User Defined field tabs, click the Customizable UDF’s button. Please refer to your VISUAL documentation for more information

  1. Click the User Defined tab to enter information in the user defined fields.

  2. Click the WBS Codes Tab to enter WBS Codes for the project. Click the Insert button to add a new code, then enter information in the following fields:

Code - Enter the code you would like to use in the Code field.

Description - Enter a description of the WBS in the Description field

CDRL - Enter a Contract Data Requirement List in the CDRL field, if necessary.

CLIN - Enter the contract line number in the CLIN field, or double-click the browse button to select a CLIN from a list.

You cannot enter a CLIN until you have set up the contract for the project in Customer Order Entry. See About Entering Contract Information in Customer Order Entry.

Customer WBS - If the customer uses a WBS code that is different from yours, enter the customer’s WBS code in the Customer WBS field.

Billable - Click the Billable arrow and select the billing option you want to use for this WBS. You can select either Billable or Non Billable.

Default Dept ID - Double-click the Default Dept ID button and select the department you want this WBS to use. VISUAL requires a Default Department ID for each WBS.

Transfer Costs - If you want to use VISUAL’s transfer costs functionality, select the Transfer Costs check box.

VISUAL does not normally allow the transfer of costs between WBS elements within the same project; when you bring inventory to a WBS and issue it to a shop order in the same project with a different WBS, the costs will not transfer. Selecting the Transfer Costs check box allows the transfer of costs from one WBS to another.

  1. Click the Budget Summary tab to view the budget totals for the project.

  2. Click the ETC Summary tab to view the ETC totals for the project.

  3. When you have finished entering information for this project, click the Save button.

The Project window creates the project and the Header card appears in the window.

After you have created a new project, you can add tasks, legs, and mate rials to meet the individual needs of your project or contract.
