Understanding Version Control

Because projects can contain large and complicated structures, VISUAL provides you with a Version Control tool. Using this tool to keep the same Project ID with the exception of the Version Control number allows you to better keep track of multiple versions of your projects.

Because VISUAL always updates the costs to Version 0 of your project, Version 0 is always the "current" version and any other versions you create are copies.

For example, using the following scenario:

You have project ANT-1/0 (version 0), and you use the New Project from option to create a copy of the project and name it ANT-1/1 (ver sion 1).

0 is always the current version and is the version to which VISUAL will send any costs when you implement the project. Version 1 is now the original.

If you then create an ANT-1/2, Version 1 is still the original and Version 2 becomes the newest copy.

For versions 0, 1, 2, and 3:

0 – Current version

3 – Newest copy

2 – First copy

1 – Original
