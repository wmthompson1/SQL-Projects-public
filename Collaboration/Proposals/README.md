## Proposal Lifecycle

```
Draft → Under Review → Approved/Declined → (If Approved) → Implementation → Completed
```

## How to Submit a Proposal

1. **Create a new file** using: `YYYY-MM-DD_proposal-name.md`
2. **Use the template** from [Templates/proposal-template.md](../Templates/proposal-template.md)
3. **Add to the table above** with status "Draft"
4. **Share with team** via Slack/email or create a GitHub issue for discussion
5. **Update status** as the proposal progresses

## Proposal Categories

- 💻 **Technology** - New tools, systems, or technical infrastructure
- 📊 **Process Improvement** - Workflow changes, efficiency initiatives
- 📈 **Business Initiative** - New business capabilities or services
- 🎓 **Training & Development** - Learning programs, skill development
- 🔧 **Quality & Standards** - Quality improvements, standardization

## Review Process

1. **Draft stage** - Author writes proposal and gathers initial feedback
2. **Review stage** - Team reviews, asks questions, provides input (1-2 weeks)
3. **Decision stage** - Decision makers approve or decline with rationale
4. **Implementation** - If approved, action items are created and tracked
5. **Closure** - Proposal marked complete with outcomes documented

## Tips for Good Proposals

✅ Be specific about the problem and solution  
✅ Include estimated costs and resources  
✅ Identify risks and mitigation strategies  
✅ Define success metrics  
✅ Keep it concise but thorough
