# Getting Started with GitHub Collaboration

**Welcome!** This guide will help you start using GitHub for collaboration, even if you've never used it before.

## What is GitHub?

Think of GitHub as a shared workspace where everyone can:
- 📝 Create and edit documents together
- 💬 Discuss ideas and proposals
- ✅ Track decisions and action items
- 📚 Build a shared knowledge base

## Your First Steps

### 1. Access GitHub
1. Go to https://github.com
2. Sign in with your Skills Inc account
3. Navigate to the SQL-Projects repository
4. Bookmark the [Collaboration folder](https://github.com/Skills-Inc-Org/SQL-Projects/tree/main/Collaboration)

### 2. Choose How to Participate

**Option A: Create/Edit Documents** (Best for: proposals, notes, glossary entries)
- Navigate to the appropriate folder
- Click "Add file" → "Create new file" or click on an existing file to edit
- Use simple Markdown formatting (see below)

**Option B: Start a Discussion** (Best for: questions, quick feedback, brainstorming)
- Click "Issues" tab at the top
- Click "New Issue"
- Choose a template (Discussion, Decision, or Proposal)

## Simple Markdown Formatting

You don't need to know code! Just use these simple tricks:

```markdown
# This creates a big heading
## This creates a medium heading

**This makes text bold**
*This makes text italic*

- This creates a bullet point
- Another bullet point

1. This creates a numbered list
2. Second item

[This creates a link](https://example.com)
```

## Common Tasks

### Adding to the Business Glossary
1. Go to `Collaboration/Business_Glossary/README.md`
2. Click the pencil icon (✏️) to edit
3. Scroll to the relevant section
4. Add your term using the provided format
5. Click "Commit changes" at the bottom

### Documenting a Decision
1. Go to `Collaboration/Decisions/`
2. Click "Add file" → "Create new file"
3. Name it: `2025-11-26_your-decision-topic.md` (use today's date)
4. Copy the [decision template](https://github.com/Skills-Inc-Org/SQL-Projects/blob/main/Collaboration/Templates/decision-template.md)
5. Fill it out and commit

### Recording Meeting Notes
1. Go to `Collaboration/Meeting_Notes/`
2. Click "Add file" → "Create new file"
3. Name it: `2025-11-26_meeting-topic.md`
4. Copy the [meeting notes template](https://github.com/Skills-Inc-Org/SQL-Projects/blob/main/Collaboration/Templates/meeting-notes-template.md)
5. Fill it out and commit

### Submitting a Proposal
**Method 1:** Create a document
- Go to `Collaboration/Proposals/`
- Create new file with template

**Method 2:** Create an issue (easier for discussion)
- Click "Issues" → "New Issue"
- Choose "💡 Proposal" template
- Fill out the form

## Tips for Success

✅ **Use clear file names** with dates: `2025-11-26_topic-name.md`  
✅ **Start small** - try editing the glossary before creating new documents  
✅ **Save often** - click "Commit changes" to save your work  
✅ **Ask questions** - create a discussion issue if you're unsure  
✅ **Preview before saving** - click the "Preview" tab to see how it looks

## Common Questions

**Q: What if I make a mistake?**  
A: Don't worry! GitHub keeps version history. We can always undo changes.

**Q: Can I edit someone else's document?**  
A: Yes! Collaboration is encouraged. GitHub tracks who made what changes.

**Q: How do I see what changed?**  
A: Click "History" button when viewing any file to see all changes.

**Q: What's a commit?**  
A: It's just GitHub's word for "save." When you commit, you save your changes.

**Q: Do I need to install anything?**  
A: No! Everything works in your web browser.
