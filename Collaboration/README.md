# Collaboration Space

Welcome to the Skills Inc collaboration space! This area is designed for team members to share ideas, make decisions, and document our shared understanding of business concepts.

## 📂 What's Here

- **[Decisions](Decisions/)** - Track important business and technical decisions
- **[Meeting Notes](Meeting_Notes/)** - Document team discussions and outcomes
- **[Business Glossary](Business_Glossary/)** - Shared definitions of terms, metrics, and concepts
- **[Proposals](Proposals/)** - New ideas and initiatives for team review

## 🚀 Quick Start Guide

### For First-Time GitHub Users

1. **Sign in to GitHub** with your Skills Inc account
2. **Navigate to this folder** in the repository
3. **Click "Add file" → "Create new file"** to start a new document
4. **Name your file** with `.md` extension (e.g., `my-proposal.md`)
5. **Write your content** using simple formatting (see tips below)
6. **Click "Commit changes"** to save

### Simple Formatting Tips

```markdown
# Main Heading
## Sub Heading

**Bold text**
*Italic text*

- Bullet point
- Another point

1. Numbered item
2. Another item

[Link text](https://example.com)
```

## 💡 How to Contribute

### Share a Proposal
1. Go to the [Proposals](Proposals/) folder
2. Create a new file: `YYYY-MM-DD_your-proposal-name.md`
3. Use the [proposal template](Templates/proposal-template.md)

### Document a Decision
1. Go to the [Decisions](Decisions/) folder
2. Create a new file: `YYYY-MM-DD_decision-topic.md`
3. Use the [decision template](Templates/decision-template.md)

### Add to Business Glossary
1. Go to [Business Glossary](Business_Glossary/)
2. Either:
   - Add to an existing glossary file, OR
   - Create a new file for your department/topic

### Record Meeting Notes
1. Go to [Meeting Notes](Meeting_Notes/)
2. Create file: `YYYY-MM-DD_meeting-topic.md`
3. Use the [meeting notes template](Templates/meeting-notes-template.md)

## 🤝 Need Help?

- **Can't find something?** Use GitHub's search (top left)
- **Want to discuss?** Create an issue with the "Discussion" label
- **Technical issues?** Contact IT Data Admin team

## 📋 Best Practices

✅ **DO:**
- Use clear, descriptive file names
- Include dates in file names (YYYY-MM-DD format)
- Write in plain language
- Link to related documents
- Update existing docs rather than duplicating

❌ **DON'T:**
- Store sensitive/confidential information (use secure channels)
- Delete other people's files without discussion
- Use special characters in file names (stick to letters, numbers, dashes, underscores)
