Navigating in the Contact Merge Utility




| | Navigating in the Contact Merge Utility | |
| On startup, a table listing all contacts is displayed in the Contact Merge Utility window. The menus, toolbar icons, merge parameters, and log file fields discussed here provide you with the ability to remove non-essential contacts from your VISUAL contact list and VISUAL CRM contact list.  The interface for this utility includes:   * Menus * Toolbar   icons * Contact   table * Contact   Merge options * Contact   Merge Log File  Menus The Contact Merge Utility menu selections are:  **File** - Allows you to start the contact merge process, refresh the window, or exit the application.  **View** Allows you to display the main toolbar and the personal menu.  **Options** Allows you to access to preference settings for sorting the columns in the contact list table.  **Help** Allows you to access to context-sensitive help. Toolbar icons The toolbar icon selections are:  **Start** Run the utility with the current preferences and merge the contact lists.  **Clear** Set all preferences to the default settings.  **Refresh** Update all of the contacts listed in the table.  **Quick Filter** Provide a list of contacts that match the filter. Click **Quick Filter** to add a top table row to the table, enter a value in for a field, and click Quick Filter again. Quick filters are additive and allow you to filter on multiple fields and continue to filter your current table view until you click Reset.  **Reset** Clear the quick filter and repopulates the table with all entries.  **View Log File** - Show the complete audit trail of the changes resulting from the merge. These changes are inclusive of modifications to the Contact ID, transactional records, specifications, the application where the changes occurred and the time and date that occurred.  **Whats This?** Provide context sensitive information for fields within the window. When the cursor becomes a cursor with a question mark, click a field and a brief definition for that item appears. Contact table The contact table lists all contacts found in the VISUAL database and the CRM database, if installed. You can filter the table by column or sort by prioritizing the contact information. If CRM is installed, the table displays contacts that are both linked and unlinked with VISUAL. General contact information is listed in the contact table along with these fields:  **Active** Indicates that the contact is identified as active when the check box is selected.  **VE ID** - Uniquely identifies each VISUAL contact with a system-generated number.  **CRM ID** - Uniquely identifies each CRM contact with a system-generated number that can be modified initially. This column is not visible if CRM is not installed. Contact Merge options Your selections define how your contact merge occurs. Verify the settings in these fields prior to performing the contact merge.  **Update details of transactional records with parent contact information** Select this check box to update all contact details for transactional records with the information of the parent contact.  **Concatenate duplicate contact(s) specifications on to the parent contact** Select this check box to append the duplicate contact specifications to the end of the parent contact specifications prior to deleting the duplicate contact transactions.  **Click the Merge tab to display these options:**  **Select Parent** Specifies the contact to add to the Selected Parent Contact panel.  **Remove Parent** - Removes the contact from the Selected Parent Contact panel.  **Parent Contact table** Designates the parent contact. When you remove a duplicate contact, the values of the parent contact are displayed in this order: VE (VISUAL Enterprise) ID; CRM ID; Contact Info - honorific, first name, middle initial, last name, position, phone, mobile, fax, and e-mail; Customer Associations; and Vendor Associations.  **Add Duplicate** Adds the selected contact from the contact list table to the Selected Duplicate Contact panel.  **Remove Duplicate** Removes the selected contact from the Selected Duplicate Contact panel.  **Duplicate Contact table** - Lists the duplicate contacts to be replaced by the parent contact. The values of each contact are displayed in this order: VE (VISUAL Enterprise) ID; CRM ID; Contact Info - honorific, first name, middle initial, last name, position, phone, mobile, fax, and e-mail; Customer Associations; and Vendor Associations.  Click the Log File tab to display these options:  **File Path** Designates the filepath for where the merge log will reside. Click the filepath browse button to open the VISUAL standard explorer window and search for the appropriate directory. The file path is saved to the local preference file VMMRGCON.INI with a variable name of Log File Directory.  **File Name** - Specifies the name of the Contact Merge log file that provides historical data of the merge. The file name is saved to the local preference file VMMRGCON.INI key name Log File Name. |
