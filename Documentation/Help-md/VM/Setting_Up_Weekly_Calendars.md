# Setting Up Weekly Calendars

1. In the Global Scheduler
   window, select Weekly Calendar from
   the File menu.

The Weekly Calendar dialog box appears.
By default, the Schedule ID and Resource ID fields default to \*\*All\*\*.

2. If you are licensed
   to use multiple sites, click the Site ID and select the site for
   which you are setting up a weekly calendar. If you are licensed
   to use a single site, this field is unavailable.
3. From the Schedule ID
   drop down box, select the schedule for which you want to establish
   the calendar.

If the weekly calendar has not been set
for this schedule, the calendar is blank.

4. Click in the 1st Shift
   Start field next to the desired day to set. Enter the time that
   the 1st shift starts.

   |  |  |
   | --- | --- |
   | TIP.gif | By default, the system assumes that the 1st shift starts in the morning (AM). Therefore, for speedy entry of time, enter the hour number. For example, if the shift starts at 7:00:00 AM, you only need to enter 7. The system fills in the rest of the entry for you automatically. |

5. Enter the number of
   hours that the resource can be used for each of the shifts.

To the Scheduler, the idea of a shift means
a contiguous period of time which a given resource is operated generally
by a given individual or team of individuals. When specifying hours
per shift, you are indicating the amount of time a given unit of capacity
is available.

If it is a machine tool, for instance, whose
availability is being defined for each shift it is operated, overlapping
is inappropriate since the machine is not actually available for the
overlapped period for one of the adjacent shifts.

If a resources unavailable on a day or during
a shift, put a zero (0) in it.

6. After you have entered
   all of the weekly scheduling information for the resource, click
   the Save button.

To delete a weekly schedule day, click the
Delete button.
