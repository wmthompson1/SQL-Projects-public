# Using List of Tests

After you have defined the quality test, assigned
the test to the operations of a parts engineering master, and reported
completed labor for the master (using Labor Ticket Entry), you are
ready to enter and analyze quality test data using Data Entry and
Analysis.

## Starting Data Entry and Analysis

Data Entry and Analysis is available from the VISUAL Main window.

Select List of Tests from the Eng/Mfg
menu.

The List of Tests window appears.

The window contains buttons for selecting test IDs, work orders
and operations with assigned tests.

The window also contains a line item table that lists the test steps
belonging to the test associated with the displayed operation. The
columns in the table are:

Step No - The Test Step number, used with
the Test ID to uniquely identify a test step.

Type - The test step type, XBar, p (proportion
nonconforming), np (number nonconforming), u (number of nonconformities
per unit) and c (number of nonconformities).

Prefix - A prefix automatically assigned
to input values to reduce the amount of typing

Central Line X - The central line of the
control chart.

LCL X - The lower control limit.

UCL X - The upper control limit.

Central Line RS - The central line of
the range or standard deviation control chart.

LCL RS - The lower control limit of the
range or standard deviation control chart.

UCL RS - The upper control limit of the
range or standard deviation control chart.

R or S - Whether Range or Standard deviation
is used in XBar control charts.

Aim - The aim or target engineering specification.

LSL - The lower specification limit.

USL - The upper specification limit.

Gauge - Description of gauge for this
test step.

Frequency - This is a description of the
type and frequency of sampling. (e.g. 3 out of 5 pieces, 3 out of
10 minutes)

Class - The defect classification, e.g.
Critical, Major, Minor, etc.

BMP - Indicates whether or not there is
a bitmap associated with this test step.

Next Sub Group - The next subgroup number
to be assigned for the test step.

Next Piece - The next piece number to
be assigned for the test step.

Next Time - Used for time base sampling
methods to determine the test steps eligibility for input.

There is a quantity data field for entering the current quantity
to report for the displayed operation. VISUAL may automatically set
this value or you can enter a value manually.

The Total Qty field displays the quantity previously reported (read
from the database) plus the current quantity to report (the value
in the Qty field).

The Hours data field is for entering the current number of hours
to report for the displayed operation. As with the Qty field, VISUAL
may automatically set this value or you can enter a value manually.
If a value is not passed by another program then the system calculates
the hours to report.

The Total Hours field displays the time in hours previously reported
(read from the database) plus the current hours to report (the value
in the Hours field).
