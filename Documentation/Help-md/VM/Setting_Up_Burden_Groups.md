# Setting Up Burden Groups

Burden groups allow you to specify several burden
categories, set up the different accounts you want to use for those
categories, and group them together. Specifying the appropriate group
allows you to consistently specify all of the associated burdens for
several projects, thus allowing you to cost certain types of projects
the same way.

When you transfer project inventory, all burden costs are assigned
to one burden category in the burden group. Costs are not recorded
against each burden category in the group.

You cannot change or delete Burden Groups if there are burden transactions
associated with them. VISUAL selects the Has History
check box when burden transactions exist for the burden group.

To set up Burden Groups:

1. From the Project menu,
   select G/L Burden Mapping Maintenance.
2. From the Maintain menu,
   select Burden Group Table.

The Burden Group Maintenance window appears.

3. Click in the Burden
   Group ID field and enter an identification for this burden group.

Press the TAB key and enter a description
for this group.

4. Click the Insert
   button.
5. Double-click the Burden Category ID row header and select the
   category you want to use.

The description automatically appears.

6. Double-click the Debit Account row header and select the debit
   account you want to use for this category.
7. Double-click the Credit Account row header and select the credit
   account you want to use for this category.
8. When you have finished
   entering categories for this group, click the Save
   button.

When you have finished entering groups,
click the Close button.

## Commercial (non-project) Burdening

VISUAL calculates burdens using the same methodology regardless
of your project or non-project (commercial) work. You may want to
set up a project type for non-project work so that VISUAL can correctly
burden your non-project costs and assign those costs to the appropriate
Burden G/L account.
