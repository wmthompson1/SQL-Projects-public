# Step Eleven

## Reconciling the Balance Sheet

VISUAL provides many tools to reconcile your commercial
transactions. The types of transactions you should reconcile are Cash,
Accounts Payable, Accounts Receivable, Inventory, Work In Process
and PO Accrual transactions.

|  |  |
| --- | --- |
| POSTIT.gif | The manufacturing reports must be run in a static environment. Any changes made to the database while you are performing the reconciliation will result in inconsistent results. |

Reconcile the following types of transactions:

Cash Transactions - To reconcile cash
transactions, compare information in the Cash Management window, the
Cash Book window, the Cash G/L Accounts
report and your companys bank statements.

Accounts Receivable - To reconcile Accounts
Receivable, compare the Aged Receivables
report to the commercial A/P account(s) in the Accounting window.

Reconciling Accounts Payable - To reconcile
Accounts Payable, compare the Aged Payables
report to the commercial A/P account(s) in the Accounting window.

Reconciling Inventory Transactions - To
reconcile inventory transactions, run the Inventory
Valuation Cost Layer Detail report and the Post
Manufacturing Journal Inventory Balance report. Compare the
results you see on these two reports to the General Ledger.

Reconciling Work In Process Transactions
- Similarly, to reconcile work in progress, compare the Work In Process
Report Posted Value to the Post Manufacturing Journal WIP Balance
report. The transactions on each of these reports should match, and
you should find the transactions represented in the General Ledger.

Reconciling PO Accrual - To reconcile
PO accrual, compare the Uninvoiced Detail Report in Payable Invoice
Entry to the Post Manufacturing Journal PO Accrual report. The transactions
on each of these reports should match, and you should find the transactions
represented in the General Ledger.
