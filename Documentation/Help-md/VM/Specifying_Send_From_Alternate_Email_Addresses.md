# Specifying Send From Alternate Email Addresses

Use the Send From Email Address for Notifications dialog box to
specify the email addresses to use as the sender for notifications.
You can specify a different address for each notification type. You
can also specify a different address for emails sent to internal recipients
and external recipients.

For example, you could specify orders@yourcompany.com as the sender
of customer order notifications. The alternate email addresses must
exist as an account and have full access in your email system. The
email addresses must also have a password for your email system.

If you do not specify an alternate address, then the email address
of the user who triggers the notification is used as the sender.

To specify an alternate sender:

1. Select Admin,
   Notification Maintenance.
2. Select Edit,
   Edit Sender Email. All notification
   types are listed in the dialog. A separate line for internal emails
   and external emails is displayed.
3. For the type of notification
   that is sent by an alternate sender, specify this information:

Email Address
Specify the email address to use. The email address must exist as
an account in your email system. The address is not validated.

Password Specify
the password that is associated with the email account. The characters
in the password are encrypted as you type.

Repeat Password
Retype the password you specified in the Password field.

4. Click Save.
