# Reports for Viewing Transactions

Use these reports to review the transactions for the
month. These reports correspond to Step One of closing procedures.

## Pack List Report

Use this report to review shipments.

1. From the Contracts Menu,
   select Shipping Entry. The Shipping
   Entry window appears.
2. From the File menu,
   select Print Pack List Report. The Pack
   List Report window appears.
3. In the Status box, choose
   Shipped.
4. In the Date box, choose
   Actual Ship Date.

The field labels at the top of the window
change to Starting Actual Ship Date and Ending Actual Ship Date.

5. In the Starting Actual
   Ship Date field, enter the first day of the month you would like
   to close. In the Ending Actual Ship Date field, enter the last
   day of the month that you would like to close.
6. In the Type box, choose
   Detailed.
7. Click Ok.
   A Detailed Pack List report appears.

The key information on the Pack List detail
report is the customer, the items shipped, the quantity and the value
of the shipment.

## Shipping/Delivery Report

Use the Shipping/Delivery Report to review shipments.

1. From the Contracts menu,
   select Shipping/Delivery Report. The
   Shipping/Delivery Report window appears.
2. In the Starting Due
   Date field, enter the first day of the period that you would like
   to close. In the Ending Due Date field, enter the last day of
   the period that you would like to close.

* In the Sequence
  box, choose the primary sort order for the report. Your choices
  are:
* Due date
* Order ID
* Customer ID
* Customer Name
* Part ID
* Supply ID

4. In the Subsequence box,
   choose the secondary sort order for the report. Your choices are:

* Due date
* Order ID
* Customer ID
* Customer Name
* Part ID
* Supply ID

5. Click the Print
   button to output the report. The report appears (the sample below
   is sequenced first by Due date, then by Order ID).

As you can see from the report, order 00013
was supposed to be delivered on 12/14/2004, but the Pack List report
indicates that the order was not delivered. Investigate the discrepancy
before closing the period.

## Customer Sales Journal Report

Use the Customer Sales Journal Report to review all
invoiced orders.

1. From the Contracts menu,
   select Customer Sales Journal. The Customer
   Sales Journal window appears.
2. In the Starting Date
   field, enter the first day of the period that you would like to
   close. In the Ending Invoice Date field, enter the last day of
   the period that you would like to close.
3. In the Sequence box,
   select the sort order for your report.
4. Select the Print
   Uninvoiced check box. This enables you to view orders that
   were supposed to ship during the month that have not been invoiced.
5. Select the Detail option
   for the report. The Detailed version includes customer order IDs
   along with other valuable information.
6. Click the Print
   button to print the report.

If there are uninvoiced transactions, you
may want to investigate why they have not yet been invoiced.

## Print One Invoice

Use the Print One Invoice window to view shipments
eligible to be invoiced.

1. From the Receivables
   menu, select Invoice Forms. The Invoice
   Forms window appears.
2. From the File menu,
   select Print One Invoice. The Print
   One Invoice window appears.

The window shows orders that have been shipped
but have not yet been invoiced. Review your records to see if an invoice
should be issued.

## Project Billing Events Report

Use this report to review project billing transactions.

1. From the Project menu,
   select Project Billing Entry. The Project
   Billing Entry window appears.
2. Select File,
   Print Project Billing Events Report.
   The Print Project Billings Event window appears.
3. In the Starting Date
   field, enter the first day of the period you want to close. In
   the Ending Date field, enter the last day of the period you want
   to close.
4. In the Show Events box,
   select Future Billing Events, Current Billing Events, and Processed
   Billing Events. Selecting all check boxes will give you
   a complete view of the billing events that are set to occur during
   the month.
5. In the Customers box,
   select All Customers. When you select
   All Customers, VISUAL selects All Orders
   in the Orders box by default.
6. Select a sort order
   for your report in the Sequence box. Your choices are by Order ID and by Customer
   ID.
7. Click Ok
   to run the report. The report outputs.

## Receiving Report

Use the Receiving Report to review purchase receipts.

1. From the Purchasing
   menu, select Receiving Report. The Receiving
   Report window appears:
2. In the Starting Date
   field, enter the first day of the period you would like to close.
   In the Ending Date field, enter the last day of the period you
   would like to close.
3. In the PO Receipt Status,
   select Scheduled.
4. Leave the fields in
   the Selection Criteria blank to return all scheduled receipts
   for the month.
5. Click the Print
   button. The Purchased Part Receiving Report Scheduled appears.
6. Run the report again,
   this time selecting Actual in the PO
   Receipt Status box.
7. Click Print.
   The Purchased Part Receiving Report Actual appears.
8. Compare the Purchased
   Part Receiving Report Actual to the Purchased Part Receiving
   Report Scheduled.

If there are scheduled deliveries that do
not appear on your actuals report, you may want to investigate the
discrepancy.

## Inventory Transactions Report

Use the Inventory Transactions report to review the
inventory transactions for the month.

1. From the Inventory menu,
   select Inventory Transaction Entry.
   The Inventory Transaction Entry window appears.
2. From the File menu,
   select Print. The Print Inventory Transactions
   window appears.
3. In the Starting Date
   field, enter the first day of the period you would like to close.
   In the Ending Date field, enter the last day of the period you
   would like to close.
4. In the Type box, use
   the check boxes to determine the types of inventory transactions
   you would like to view.
5. Select Receipts,
   Issues/Shipments, and Adjustments
   to view all inventory transactions.
6. In the Sequence box,
   select a sort order for your report. Your choices are:

* Transaction
  ID
* Part ID
* Customer Order
  ID
* Purchase Order
  ID
* Work Order
  ID
* Warehouse/Location
* Warehouse/Location/Part
  ID

For projects, you may want to sort the report
by Warehouse/Location to view transactions by project (since project
warehouse IDs are identical to project IDs).

7. Select Include
   Obsolete Parts to include parts that you may have recently
   put out of service.
8. Click Ok.
   The report appears.

## Labor Tickets

Use the Labor Tickets report to review the labor tickets
entered for the month.

1. From the Eng/Mfg menu,
   select Labor Ticket Entry. The Labor
   Ticket Entry window appears.
2. From the File menu,
   select Print Summary. The Print Summary
   window appears.
3. In the Starting Date
   field, enter the first day of the period you would like to close.
   In the Ending Date field, enter the last day of the period you
   would like to close.
4. Leave the Employee ID
   fields blank to return information on all employees. This gives
   you a complete analysis of the labor tickets submitted during
   the period.
5. Make selections in the
   following boxes as appropriate:

Week Beginning box
Select Selected Starting Date to set the first day of the first
week in the report to the starting date specified in the Starting
Date field. Select 1st Transaction Date Found to set the first day
of the first week in the report to the date of the first transaction
recorded.

Major Sequence
Select By Employee ID to sort the report by employee ID. Select
None to sort the report solely by the minor sequence. We recommend
selecting By Employee ID.

Minor Sequence
Select the secondary sort for the report in the Minor Sequence box.
Your choices are:

* By Date
* By G/L Account
  ID
* By Resource
  ID
* By Work Order
  ID
* By Department
  ID

We recommend selecting By Work Order ID
so you can trace the labor tickets to a particular work order.

Display Date
Select Display by Shift Date to show the shift date on the report.
Select Display by Transaction Date to use the transaction date.

Determine Overtime
Select the Determine Overtime check box to compute overtime wages
on the report.

Include Vac/Hol/Sick
Select this check box to include paid leave on the report.

6. Click Ok.
   The report outputs.

## A/R Invoice Register

Use the A/R Invoice Register to review A/R transactions.

1. From the Receivables
   menu, select Invoice Entry. The Accounts
   Receivable Invoice Entry window appears.
2. From the File menu,
   select Print Invoice Register. The Print
   Invoice Register window appears.
3. In the Beg Invoice Date
   field, enter the first day of the period you would like to close.
   In the End Invoice Date field, enter the last day of the period
   you would like to close.
4. In the Include box,
   select the types of invoices to include in the report. Your choices
   are:

* Unpaid
* Partially
  Paid
* Paid in full
* Void/Cancelled
* Credit Memo
* Unapplied
  Cash

You may find it useful to run the report
type by type.

5. In the Sort Order box,
   arrange the sort order for the report by using the up and down
   arrow buttons. For example, to sort the report first by Invoice
   Date, select Invoice Date, then click the Up Arrow button to move
   Invoice Date to the top of the list.
6. In the Totals box, select
   the types of subtotals and totals you would like to view. You
   can select none, one or all of the following:

* Customer Subtotal
* Currency Subtotal
* Entity Subtotal
* Grand Total

7. Click Ok.
   The report outputs.

## Accounts Receivable Cash Receipts

To review the cash receipts for the period you would
like to close, run the Cash Receipts report.

1. From the Receivables
   menu, select Cash Application. The Cash
   Application window appears.
2. From the File menu,
   select Print Cash Receipts. The Print
   Cash Receipts window appears.
3. In the Starting Date
   field, enter the first day of the period you would like to close.
   In the Ending Date field, enter the last day of the period you
   would like to close.
4. In the Sort by box,
   select Payment ID to view the report
   by payment ID, or Deposit ID to view the report by Deposit ID.
5. The Print Active Payments
   check box is selected by default. If you select Print Void Payments,
   you can clear the Print Active Payments
   check box. Select the Summary check box if you would like to view
   a summary report; we recommend leaving the check box blank to
   view a detailed report.
6. Click Ok.
   The report outputs.

## Uninvoiced Receivers Report Detailed

To make sure that invoices have been entered for all
receivables for the month, run the Uninvoiced Receivers report.

1. From the Payables menu,
   select Invoice Entry. The Accounts Payable
   Invoice Entry window appears.
2. From the File menu,
   select Print Uninvoiced Receivers Report Detailed.
   The Print Uninvoiced Receivers Report Detailed report window appears.
3. In the Starting Date
   field, enter the first date of the period you would like to close.
   In the Ending Date field, enter the last date of the period you
   would like to close.
4. In the Status box, select
   the types of invoices you would like to view in the report. For
   the most complete report, select all options. Your options are

* Released
* Closed
* Cancelled

5. In the Sequence box,
   select the order for the report. You can sort the report by Purchase Order ID or Vendor
   ID.
6. In the Type box, select
   Summary to view a summary report or Detailed to view a detailed
   report. We recommend running a detailed report. The detailed report
   shows purchase order line detail.
7. Click Ok.
   The report outputs.

## Revenue Recognition Transactions Report

The Revenue Recognition Transactions report shows
details of your revenue recognition transactions for projects.

1. From the Project menu,
   select Revenue Recognition. The Revenue
   Recognition window appears.
2. From the File menu,
   select Print Transactions. The Print
   G/L Transactions window appears.
3. In the Print Selection
   box, select By Accounting Period.
4. In the End Date of Acct
   Period field, enter any day within the period you would like to
   close. VISUAL determines the end date of the period and populates
   the field.
5. In the Transactions
   box, select the type of transactions you would like to view. We
   recommend running the report twice: once for unposted transactions
   and once for posted transactions.
6. In the Sequence box,
   select the sort order for the report. Your choices are by Transaction ID, G/L Account
   ID, or by Type, Project
   ID, Transaction ID.
7. Clear the Current
   User Only check box to view transactions from all users
   in the report.
8. Click Ok.

## Deferred Revenue Report (Commercial Revenue Recognition)

The Deferred Revenue report shows revenue recognition
transactions for the period and the Milestone Status Report to review
progress billings that are ready to be triggered

1. From the Receivables
   menu, select Progress Billing Entry.
   The Progress Billing Entry window appears.
2. From the File menu,
   select Print Deferred Revenue Status Report.
   The Print Deferred Revenue Status Report window appears.
3. In the Starting Date
   field, enter the first day of the period you would like to close.
   In the Ending Date field, enter the last day of the period you
   would like to close.
4. In the Customers box,
   select All Customers to view information
   for all customers. This provides a complete view of the period.
5. In the Orders box, select
   All Orders to view information for all
   orders. This provides a complete view for the period.
6. In the Sequence box,
   select a sort order for the report. Your choices are by Order ID or by Customer.
7. In the Type box, select
   Detail. The detailed report provides
   more granularity to the information.
8. Click Ok.
   The report outputs.

If there had been any commercial revenue
recognition for the period, it would have appeared in the Recognized
Rev column.

## Milestone Status Report

Use the Milestone Status Report to review progress
billings that are ready to be triggered

1. From the Progress Billing
   Window, select File, Print
   Milestone Status Report. The Print Milestone Status Report
   window appears.
2. In the Starting Date
   field, enter the first day of the period you would like to review.
   In the Ending Date field, enter the last day of the period you
   would like to review.
3. In the Show Events box,
   select Ready to Trigger to review events
   that are ready to be processed. If you would also like to see
   the events that have already been triggered during the time period,
   select the Already Triggered check box
   as well.
4. In the Customers box,
   select All Customers to review a comprehensive
   set of data.
5. In the Sequence box,
   select the sort order for the report. You can choose to sort the
   report by Order ID or Customer
   ID.
6. Click Ok.
   The report outputs.
