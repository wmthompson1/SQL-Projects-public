# Reconciling the Balance Sheet

VISUAL provides many tools to reconcile your commercial
transactions. The types of transactions you should reconcile are Cash,
Accounts Payable, Accounts Receivable, Inventory, Work In Process
and PO Accrual transactions.

|  |  |
| --- | --- |
| POSTIT.gif | The manufacturing reports must be run in a static environment. Any changes made to the database while you are performing the reconciliation will result in inconsistent results. |

Reconcile the following types of transactions:

Cash Transactions - To reconcile cash
transactions, compare information in the Cash Management window, the
Cash Book window, the [Cash
G/L Accounts](Cash_G_L_Accounts_Report.md) report and your companys bank statements.

Accounts Receivable - To reconcile Accounts
Receivable, compare the [Aged Receivables](A_R_Aged_Receivables.md)
report to the A/R account(s) in the Accounting window.

Reconciling Accounts Payable - To reconcile
Accounts Payable, compare the [Aged
Payables](A_P_Aged_Receivables.md) report to the commercial A/P account(s) in the Accounting
window.

Reconciling Inventory Transactions - To
reconcile inventory transactions, run the [Inventory
Valuation - Cost Layer Detail](Inventory_Valuation_Report.md) report and the [Inventory
Balance Report](Inventory_Balance_Report.md). Compare the results you see on these two reports
to the General Ledger.

Reconciling Work In Process Transactions
- Similarly, to reconcile work in progress, compare the [Work
In Process Report - Posted Value](Work_In_Process_Posted_Value.md) to the Post Manufacturing Journal
[WIP Balance](Running_WIP_Balance_Inquiry.md) report.
The transactions on each of these reports should match, and you should
find the transactions represented in the General Ledger.

Reconciling PO Accrual - To reconcile
PO accrual, compare the [Uninvoiced
Detail Report](Uninvoiced_Receivers_Report_Detailed.md) in Payable Invoice Entry to the Post Manufacturing
Journal [PO Accrual](PO_Accrual_Report.md) report. The
transactions on each of these reports should match, and you should
find the transactions represented in the General Ledger.

Three reports are particularly helpful when reconciling project
transactions: the [EAC/POC](EAC_POC_Report_helpful.md)
Report, the Project WIP Balance report and the [Project
Billing Events](Project_Billing_Events_Report_Helpful.md) report.

Use the [EAC/POC Report](EAC_POC_Report_helpful.md)
to review and analyze the actual amounts and cost of goods sold by
project. Print the EAC/POC Report by selecting File, Print EAC/POC
from the Project Summary window.

Use the [Project WIP Balance
Report](Running_WIP_Balance_Inquiry.md) to reconcile project WIP balances. Project WIP balances
are made up of project inventory, plus direct project labor, plus
other direct charge amounts, plus general ledger adjustments, plus
A/P invoice charges, minus amounts moved to COGS through revenue recognition.
Print the Project WIP Balance Report by selecting Reports, Project
WIP Balances from the Post Manufacturing Journals window.

Use the [Project
Billing Events Report](Project_Billing_Events_Report_Helpful.md) to reconcile the customer order value to
the contract value of your projects. You can also use this report
to reconcile deferred revenue. Print the Project Billing Events Report
by selecting File, Print Project Billing Events from the Project Billing
window.
