# Adding Parts Without Part IDs

You can set up pricing tables for parts with no part
IDs. Only the Vendor Part ID is necessary to use this information
to place a purchase order. This is helpful if you buy parts for a
specific job for which your company has not assigned part IDs.

By using the following procedure, you can enter these parts into
the system, thereby enabling the purchasing department to order them
using the vendor part ID. This function is not specific to the current
part in the Part Maintenance window; you can use it at any time.

1. Click the Clear
   button to clear the window.
2. From the Part Maintenance
   Maintain menu, select Vendor Parts Supplied.

The Vendor Parts Supplied window appears.

After you specify a vendor, the vendor part
IDs that are currently assigned to this part appear in the dialog
box. If no Vendor Part IDs are assigned, the table is blank.

3. Enter the [Vendor
   ID](javascript:TextPopup(this)) for the new vendor in the Vendor ID field.

   Specify or select the identification code for the Vendor
   whose part information you want to view. You can enter the
   Vendor ID manually, or select it from the browse by clicking
   the Vendor ID button. All vendor part IDs currently assigned
   to the vendor appear in the table. If no vendor part IDs exist,
   the table is blank.

The Vendor ID must exist in the Vendor tables
before you can use it. The table shows all parts for the selected
vendors that have a specified Vendor Part ID. This includes regular
inventory parts that were also given a Vendor Part ID using Vendors
Supplying this Part.

You can call up a table of available vendors
by selecting the Vendor ID button, then
selecting the desired vendor for use.

4. Click the Insert
   button and enter the [Vendor
   Part ID](javascript:TextPopup(this)).

   This column contains a list of all the Part IDs assigned
   to the current vendor.

|  |  |
| --- | --- |

If you exit the vendor part ID table to enter price information, you must select the SAVE button before returning to enter more vendor part IDs. Failure to do so causes the information you entered outside the vendor part ID table to be lost. |

5. Enter the prices and
   quantity breaks for the part in the price break table if desired.
   This information is used when writing a purchase order, but is
   not required.
6. Enter the purchase U/M
   for the part. The U/M must exist in the unit table before you
   can use it. Select the Purchase U/M button to select a U/M from the units
   table.
7. Enter the date for the
   prices quoted in the Quote Date field. By default, the date automatically
   reflects the current date.
8. Click the Save
   button to save the information.
9. You can update the price
   information for a vendor part at any time by selecting the part
   from the table, modifying the price fields, and pressing Save.
10. Click the Close
    button to close the Vendor Parts Supplied dialog box.
11. The part information
    is now accessible to purchase order entry.
