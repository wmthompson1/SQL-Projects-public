# Receiving Returned Shipments from Customers with RMA Requirements

Some customers require an RMA before they can return
shipments, partially or whole, for repair, replacement, or credit.
If a customer needs an RMA, VISUAL alerts you with a note box reminding
you that a customer needs an RMA before you can accept returned quantities.

|  |  |
| --- | --- |
| POSTIT.gif | Specify if a customer must generate an RMA for returned shipments in Customer Maintenence. With the appropriate customer in the window, select the RMA required for return box in the Credit Limit tab. |

1. With the appropriate
   order line in the Customer Order Shipment Returns dialog box,
   click the RMA IDbutton.

A list of all the RMAs in your database
appears.

|  |  |
| --- | --- |
| IMPORTNT.gif | There must be an RMA in your database for this return before you can receive any returned shipments from a customer. |

2. Select the RMA that
   you created for the return of this shipped quantity.

The RMA ID appears in the field to the right
of the RMA ID button.

VISUAL fills the Return
Quantity Authorized column in the line item table with the
maximum quantity that the customer can return.

3. Enter the quantity that
   the customer is returning in the Quantity Returned column.
4. Click Save
   to receive the returned quantity.
