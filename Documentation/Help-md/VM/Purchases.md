# Purchases

Costing tools check purchase orders against the appropriate
detail and distribution (posted) transactions. Any discrepancies between
the detail and posted transaction totals are identified by a note
in the Remarks field, furthermore, the values in the rows do not match.

When you access the Costing Tools window, the Posting Date defaults
to the last day of the prior period. If the starting and ending ID
fields are blank when you click the Check
button, apparent database inconsistencies may appear. However, these
"inconsistencies" are a result of all transactions being
accounted for, but only the details and posting totals being displayed
in their respective columns.

## Verifying Costing Records

1. Enter the Starting and
   Ending Order IDs.
2. Click the Check
   button.

This populates the detail area with any
transactions within the range of orders. Any cost inconsistencies
are flagged by a note in the Remarks column.

If there are any cost inconsistencies, you
may correct them in one of two possible methods; either immediately
or the next time you run the Costing Utilities.

## Correcting Costs Immediately

1. Select the appropriate
   lines to be corrected. Click in the corresponding row headers.

This places a check mark (3) in the row
headers, indicating the lines to be changed.

2. Click the Post
   button.

Costing Tools makes any necessary adjustments
to the costs.

## Correcting Costs the Next Time You Run Costing

1. Select the appropriate
   lines to be corrected. Click in the corresponding row headers

This places a check mark (3) in the headers,
indicating the lines to change.

2. Click the Set
   Posting Candidate button.

This changes change the value in the Posting
Candidate (P/C) column from N to Y.

If VISUAL determines it is appropriate to
do so, the adjustment will be made the next time you run Costing Utilities.
