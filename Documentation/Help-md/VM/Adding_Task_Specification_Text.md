# Adding Task Specification Text

As mentioned in the [previous
procedure](Viewing_Requisition_Tasks_.htm), it is possible to add specification text to requisition
task sequences. It is not, however, possible to modify existing specification
text, which may include auto-generated text inserted on the changing
of a task status from Pending to Completed.

1. To view or modify a
   sequences specifications, click the Task Specifications
   button, double-click the line, or select Specifications
   from the Edit menu.

The Task Specifications dialog box appears.

Each specification appears with a date/time
stamp, the appropriate User ID, and explanatory text.

2. To add a new specification,
   click the Insert button.

The Add Comment dialog box appears.

The name of the User currently signed into
VISUAL appears in the User Name field; the current system date appears
in the Date field.

3. Place your cursor in
   the field and enter the appropriate specification text.
4. Click Save
   to commit the text to the task.

The Add Comment dialog box is closed and
you are returned to the Task Specifications dialog box, where it displays
the new comment, prefaced by the User ID and date and time of entry.

5. Click Save
   to return to the Tasks dialog box.

A different Tasks Specifications button
is used on the toolbar if the task has specification text attached
to it.

This specification button indicates that
the task has a specification.

This specification button with the diagonal
line through it indicates the task does not have a specification.
