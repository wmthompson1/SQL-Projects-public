# Linking and Embedding

Linking and Embedding is a very powerful tool provided
by the Windows environment. By using other applications you can add
more informational capabilities to Infor VISUAL.

Infor VISUAL is a Client application. This means that it can accept
information from other applications but cannot supply information
(for linking and embedding) to other applications. An application
that can supply information is called a Server. This capability allows
you to attach a drawing or document to an item without the need to
copy or input all the information again.

To change information that has been linked or embedded, you do not
need to delete it and copy it over again. The linked or embedded object
can launch the Server application with the information loaded into
it ready for editing.

When you embed or link information, you are inserting the information
from one document (the source document) into a document in a different
application (the destination document). Embedding is similar to copying
using the Clipboard but with one big difference. The embedded object
has the ability to call the server application and allow you to make
changes to the embedded object. This means that you no longer have
to switch between applications when cutting and pasting.

Embedding - When you embed information,
VISUAL copies the information in the source document into the destination
document. You no longer have a connection to the source document,
only the source application. Therefore, when you edit the embedded
object, the source document remains unchanged. For example, if you
embed a drawing into a product, then change the embedded drawing from
inside the product, the drawing in the source document is not affected.

Linking - Linking does not make a copy
of the source information, but creates a reference (link) to the source
document. This can be both an advantage and disadvantage. The advantage
is that you only have one copy of the source document. This makes
it easier to control drawing configuration and takes up less disk
space. The disadvantage is that if somebody changes the source document,
your linked information is going to change.

A complete discussion of Embedding and Linking is contained in the
Windows Users Manual.
