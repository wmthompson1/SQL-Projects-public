# Printing the Security Report

You can print a report of the modules to which a valid
VISUAL user has access. These reports are useful to System Administrators
who need to be able to keep track of user access. In assigning tasks,
a System Administrator can delegate more effectively by reading this
report before they makes any assignments.

1. From the Security menu,
   select the Print Security Report option.
2. In the Report by section,
   select one of the following options:

User Select
this option to view the report for User IDs.

Profile Select
this option to view the report for Profiles.

3. Specify the following:

Starting and Ending ID
fields In the Starting Profile or Starting User ID field,
click the browse button and select the first User ID or Profile ID
you would like to view in the report. In the Ending Profile/User ID
field, select the last ID you would like to view in the report. The
system will display information for the IDs you specify in the Starting
and Ending ID fields, plus all IDs between them. If you would like
to view all IDs, leave the Starting and Ending ID fields blank.

Starting Module ID and
Ending Module ID fields In the Starting Module ID field,
specify the first VISUAL module you would like to view in the report.
In the Ending Module ID field, specify the last module you would like
to view in the report. Use the actual program file name without the
exe extension. For example, to specify Sales Order Entry, specify
VMORDENT. The system will display information for the modules you
specify in the Starting and Ending Module ID fields, plus any module
that falls between the two alphabetically. Leave both fields blank
to view on modules in the report.

4. Select how you want
   VISUAL to output the report:

Print Select
this to view a print preferences dialog box after clicking Ok.

View Select
this to view the report on your screen after clicking Ok.

File Select
this to save the report to a file after clicking Ok.

E-mail If you
want to e-mail the report, select this option. VISUAL generates your
report in a Rich Text Format (.RTF) and attaches it to the e-mail.
If you want to attach a PDF of the report, select the PDF button.

5. In the Sequence section,
   select the order in which you want VISUAL to populate the report.
   You can select:

By User/Profile ID
The system displays the users or profiles in alphabetical order.
The field label changes based on your selection in the Report by field.

By Module ID, User/Profile
ID The system displays modules in alphabetical order. Within
the modules, the system displays the users or profiles in alphabetical
order.

By User/Profile ID, Permission
Type The system displays the user or profiles in alphabetical
order. For each user or profile, the system displays each permission
type in alphabetical order.

6. If you would like to
   view only want only the components to which users have restricted
   access to appear, select the Show Restricted
   Access Only check box.
7. Click Ok.
   VISUAL generates the report.
