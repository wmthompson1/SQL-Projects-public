# Requisition Maintenance Settings

In the Requisition Maintenance section, click in each
of the Approval Label text boxes and enter the name of the label you
want to appear on the Approval tab of the Purchase Requisition Maintenance
window.

If you want VISUAL to generate all of the tasks for your approval
groups simultaneously, select the Generate all
Tasks Simultaneously check box.

If you want to require the use of passwords for the fields to which
you have added security, select the Passwords Required
for Secured Fields check box.

For more information, refer to Building Approval Groups.

## Setting Up Rejection Codes:

Before creating any new purchase requisitions, add
rejection codes to the database. Rejection codes help you classify
rejected purchase requisitions task/ECN tasks by any quantifiable
measure you choose to impose.

|  |  |
| --- | --- |
| POSTIT.gif | Purchase Requisition Entry and ECN Entry share a common Rejection Code table. This feature is available from both the ECN tab and the Default tab in Application Global maintenance. |

1. From the main VISUAL
   menu, select Accounting Entity Maintenance
   from the Admin menu.

2. Click the Defaults
   tab.

Requisition controls appear in the Requisition
Maintenance section.

3. Click the Rejection
   Codes button.

The Maintain Task Rejection Codes dialog
box appears.

4. Click the Insert
   button to add a new rejection code.
5. Enter information as
   follows:

Code - Enter an
alphanumeric identifier for the rejection code. VISUAL supplies you
with several predefined codes.

Description -
Enter a description for the code.

6. Repeat this procedure
   for each code you want to add.
7. Click the Save
   button to commit the rejection codes to the database.

Click the Close
button to return to the main window.
