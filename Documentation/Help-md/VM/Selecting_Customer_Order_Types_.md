# Selecting Customer Order Types

1. With the customer for
   whom you want to select an order type, click the Order
   Mgt. tab.

The only fields available in the Order Mgt
tab are the Order Type field and Expire Date field.

The Order Mgt tab contains these other read-only
fields:

Shipto ID - The
customers default ShipTo ID as you entered it in Customer Maintenance.
See the earlier section for more information.

Warehouse ID -
The customers primary warehouse ID as you entered in Customer Maintenance.
See the earlier section for more information.

Total Weight U/M
- The Order Management default weight unit of measure. If applicable,
set this default in Application Global Maintenance.

2. From the Order Type
   list box, select the appropriate Order type.

Select from the following:

Regular - A typical
customer order, one that is "regular" for a customer; no
special requirements or dispensations are involved. This is the default.

Back Order - The
process used by a company when a customer orders inventory that is
not available. The company fills the order when the inventory becomes
available.

Drop Ship - A
situation when an equipment operator deposits a trailer at a facility
for it to be loaded or unloaded, but does not wait for the task to
be completed.

Emergency - As
differentiated by an order type, an emergency order is processed in
an expedited fashion.

FIRST - Items
received first are to be picked and shipped first. The general term
for a FIFO data structure is a Queue.

Flow Through -
A distribution process involving direct linkage between specific receipts
of inventory and shipments.

International
- An international order, meaning that you must address international
shipping and tax issues.

SECOND - A secondary
classification to FIRST. Items received second are to be picked and
shipped second.

Will Call - You
will call the customer when you have completed production.

3. Continue to enter customer
   order information.
