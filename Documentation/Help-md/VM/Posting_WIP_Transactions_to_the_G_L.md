# Posting WIP Transactions to the G/L

After you have verified that your transaction entries
are correct, you must post them to the General Ledger. DO NOT post
to the General Ledger if any transactions are incorrect or need to
be modified or voided.

1. From the File menu,
   select Post to G/L or click the [Post to G/L](javascript:void(0);) button on the main toolbar.

The Post to G/L dialog box appears.

2. Enter the following
   information in the dialog box.

Site ID - Click
the Site ID arrow and select a site to use.

Batch Date - The
Batch date field automatically populates with the current system date.
To change the date, click the Calendar button,
or enter the date in the field provided.

All un-posted vouchers dated up to and including
the Batch date are posted.

Accounting Period
- VISUAL automatically determines the month and year when you tab
out of the Batch Date field.

Current User Only
- Select this checkbox to post only your transaction.

Leave the Current User Only option unchecked
to post all transactions.

Click Start to
begin the posting process.

A dialog box appears asking if you want
to preview the posting to the General Ledger.

3. Click Yes
   to display the Print G/L Transactions dialog box to activate the
   print request.

You can direct the output to the screen,
a printer or an external file.

After outputting the transactions, click
Close to disable the dialog box.

Click No while
on the Print G/L Transactions dialog box to proceed with the posting
without viewing, printing, or outputting the transactions first.

VISUAL re-displays the Post to General Ledger
dialog box.

4. Click Yes
   to post the transactions or click the No button to stop the process.
   Stopping the process allows you to go back and make corrections,
   if necessary.

VISUAL informs you of which batches it processes.

If VISUAL detects an error during the posting,
a message displays. You can return to the WIP Maintenance window and
resolve the problem. If the posting is successful, a dialog box appears
asking if you want to print the batches. Either way, the WIP transactions
are posted to the General Ledger.

After posting a WIP transaction to the general
ledger, VISUAL selects the Posted check box in the main WIP Maintenance
window.
