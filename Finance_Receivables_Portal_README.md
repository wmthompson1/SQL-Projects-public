# Finance & Receivables SQL Request Portal

## Purpose

This is the **single entry point** for Finance and Receivables team members to request SQL reports and data queries from IT Data Admin.

## Files

- **`Finance_Receivables_Portal.html`** - The landing page employees should bookmark
- **`.github/ISSUE_TEMPLATE/sql-request.yml`** - The structured request form

## Setup Instructions

### 1. Host the HTML File

**Option A: GitHub Pages (Recommended)**
1. Push this file to the repository
2. Enable GitHub Pages in repo settings
3. Share the URL: `https://skills-inc-org.github.io/SQL-Projects/Finance_Receivables_Portal.html`

**Option B: Local Network**
1. Copy `Finance_Receivables_Portal.html` to a shared network drive
2. Share the file path with the team

**Option C: Email/SharePoint**
1. Email the HTML file to team members
2. They can save it locally and open in browser

### 2. Team Instructions

**For Finance/Receivables Team:**
1. **Bookmark the portal page** for easy access
2. Click "📝 Submit SQL Request"
3. Fill out the simple form describing what you need
4. Submit and wait for notification from IT Data Admin

**For IT Data Admin (wmthompson1):**
1. Receive notification when new SQL request is submitted
2. Review the request details in GitHub Issues
3. Use Copilot to help generate the SQL query
4. Respond with results or questions directly in the issue
5. When complete, attach the report and close the issue

## Workflow

```
Employee → Bookmarked Portal → Click "Submit SQL Request" 
   ↓
GitHub Issue Created with Details
   ↓
wmthompson1 Notified → Reviews Request
   ↓
Uses Copilot to Generate SQL → Tests Query
   ↓
Posts Results → Employee Gets Notification
   ↓
Issue Closed or Made into Recurring Report
```

## Benefits

✅ **Single, simple entry point** - employees bookmark one page  
✅ **Structured requests** - form ensures you get all needed info  
✅ **Tracked history** - all requests logged in GitHub Issues  
✅ **Notifications** - automatic alerts when request is fulfilled  
✅ **Searchable** - past requests easy to find and reference  
✅ **No training needed** - simple form, plain language

## Example Usage

**Employee sees the portal, clicks Submit SQL Request, and writes:**
> "What are the receivables from July 1?"

**You receive the issue notification, respond:**
> "I've run the query. See attached Excel file with receivables from July 1, 2025 to present. Total outstanding: $XXX,XXX. Let me know if you need any adjustments to this report."

**Employee gets notification, downloads file, issue resolved!**
